# Анализ модуля загрузки с сайта и шаблон XML

## Что сделано

1. Установлен и запущен MCP-сервер `mcp-bsl-platform-context` (справка по платформе 1С).
2. Проанализирован модуль `модуль загрузки с сайта.txt` (49 207 строк).
3. Выделена процедура загрузки XML — `ЗагрузитьДанныеНаСервере` (строка 32851).
4. Подготовлен корректный шаблон `order_template.xml` и пример `order_example.xml` для загрузки заказа.

## Установленный MCP-сервер

- Репозиторий: `https://github.com/alkoleft/mcp-bsl-platform-context`
- Собран JAR: `mcp-bsl-platform-context/build/libs/mcp-bsl-context-0.3.2.jar`
- Запущен в режиме SSE на порту `8081`.
- Проверена работоспособность: инициализация MCP, список tools выполняются.
- **Ограничение:** на текущей установке 1С отсутствует файл справки `shcntx_ru.hbk`, поэтому поиск по синтакс-помощнику возвращает ошибку инициализации индексов. Для полноценной работы нужен полный дистрибутив 1С:Предприятие с толстым клиентом, где есть `shcntx_ru.hbk`.

## Ключевые находки из модуля

### Процедура загрузки

```bsl
Процедура ЗагрузитьДанныеНаСервере(АдресФайлаВоВременномХранилище, Расширение)
```

- Открывает XML через `Новый ЧтениеXML`.
- Читает шапку заказа.
- При встрече `<productRow>` добавляет строку в ТЧ `Объект.Заказы`.
- При закрытии `</Tabl>` сбрасывает флаг новой строки.

### Шапка заказа (элементы верхнего уровня)

| XML-элемент | Куда загружается | Примечание |
|-------------|------------------|------------|
| `numberDate` | `Объект.ДатаЗаказа__Загрузка` | Преобразуется из `ДД.ММ.ГГГГ` в `ДД.ММ.ГГГГ 0:00:00` |
| `numberOrder` | `Объект.НомерЗаказа__Загрузка` | Строка |
| `INN` | `Объект.ИНН_Загрузка` | Используется для поиска контрагента |
| `customer` | `Объект.Контрагент_Загрузка_Строковая` | Наименование |
| `email` | `Объект.Почта` | |
| `phone` | `Объект.Телефон` | |
| `contact` | `Объект.Контакт` | |

### Строка заказа (`<productRow>` внутри `<Tabl>`)

| `connection_0` | `СоединениеВоздуховода_c0`, `СоединениеВоздуховода_r0` | Берётся `Лев(значение, 1)` |
| `connection_1` | `СоединениеВоздуховода_c1`, `СоединениеВоздуховода_r1` | Берётся `Лев(значение, 1)` |
| `connection_2` | `СоединениеВоздуховода_c2`, `СоединениеВоздуховода_r2` | Берётся `Лев(значение, 1)` |

### Формат характеристики

Парсер `Определение_Значения` ищет подстроку вида `ПАРАМЕТР<значение>ПАРАМЕТР_`:

| Параметр | Пример значения | Фрагмент строки |
|----------|-----------------|-----------------|
| `D0` | 160 | `D0160D0_` |
| `L0` | 1250 | `L01250L0_` |
| `A0` | 300 | `A0300A0_` |
| `B0` | 200 | `B0200B0_` |

Примеры:
- круглый воздуховод D=160, L=1250: `D0160D0_L01250L0_`
- прямоугольный воздуховод 300×200, L=1000: `A0300A0_B0200B0_L01000L0_`

### Применимые параметры по продуктам

Набор параметров и соединений для каждого продукта настраивается в справочнике `асГруппыПродукции` → табличная часть `ГруппыПродукции`.

| Продукт | Артикул | Применимые параметры | Применимые соединения |
|---------|---------|----------------------|----------------------|
| Воздуховод круглого сечения прямошовный | `1-1-1` | `D0`, `L0` | `c0`, `c1` |
| Воздуховод круглого сечения спирально-навивной | `1-1-2` | `D0`, `L0` | `c0`, `c1` |
| Воздуховод прямоугольного сечения | `1-2-1` | `A0`, `B0`, `L0` | `r0`, `r1` |
| Фланец (кр) | `12-1-3` | `D0`, `p` | — |

Если передать параметр, неприменимый для продукта (например, `L0` для фланца или `A0`/`B0` для круглого воздуховода), он будет проигнорирован или приведёт к некорректному обозначению.

### Материал

В `<material>` нужно передавать **код типа материала** (одна цифра): `1` — оцинкованная сталь, `2` — нержавеющая сталь, `3` — чёрная сталь. **Артикул материала, префикс артикула и код справочника `асМатериалы` (например, `000000002`) не подходят**, потому что далее вызывается:

```bsl
асОбщийМодуль.НахождениеМатериалаПоТипу(Стр.Артикул_Материала, Стр.Продукт);
```

После этого `ЗаказыtПриИзмененииНаСервере` ищет материал того же родителя с толщиной из `<thickness>`.

Примеры корректных значений `<material>`:

| Значение | Описание |
|----------|----------|
| `1` | Оцинкованная сталь. **Рекомендуется для тестовых позиций 1-1-1, 1-1-2, 1-2-1, 12-1-3** |
| `2` | Нержавеющая сталь |
| `3` | Чёрная сталь |
| `01-02-01-01` | Префикс артикула «Рулон оц.(08ПС)» (не подходит для загрузки — функция ожидает цифровой код типа) |
| `01-02-01-01-0.70` | Полный артикул с толщиной (не подходит для загрузки) |
| `01-01-03-17` | Тип «черный лист Ст-3» |
| `01-01-03-17-0.90` | Конкретный материал толщиной 0,90 мм |
| `01-01-01-01` | Тип «оцинкованный лист» |

> **Примечание:** в реальном заказе `000000839`, созданном через интерфейс, для воздуховодов использовался рулон оцинкованный толщиной 0,70/0,90 мм. Однако при загрузке из XML функция `НахождениеМатериалаПоТипу` ожидает **цифровой код типа материала** (`1`, `2`, `3`), как в тестовой выгрузке модуля «Ввод нового заказа». Если передать артикул или префикс артикула, материал не находится, и толщина принудительно сбрасывается на `0.50` (корректировочная). Чёрный лист `01-01-03-17` для этих продуктов не подходит.
>
> **Проверено:** при загрузке `order_example.xml` в спецификацию `000000841` материал и цены подобрались корректно для всех 4 позиций при `<material>1</material>` и `<thickness>0.7</thickness>`.

### Соединения

Для каждого `<connection_N>` берётся только первый символ (`Лев(значение, 1)`), затем вызывается:

```bsl
асОбщийМодуль.НахождениеСоединениеПродуктаПо_Id_c(Id_N); // круглые
асОбщийМодуль.НахождениеСоединениеПродуктаПо_Id_r(Id_N); // прямоугольные
```

Допустимые идентификаторы (по данным справочника `асСоединениеВоздуховодов`):

**Круглые (`_Id_c`):**

| ID | Соединение | Код справочника |
|----|------------|-----------------|
| `1` | Бандаж | `000000001` |
| `2` | Без соединения | `000000002` |
| `3` | Ниппель | `000000003` |
| `4` | Фланец (кр) | `000000004` |

**Прямоугольные (`_Id_r`):**

| ID | Соединение | Код справочника |
|----|------------|-----------------|
| `5` | Фланец (пр) | `000000005` |
| `6` | Шина / уголок | `000000006` |
| `7` | Рейка | `000000007` |
| `8` | Без соединения | `000000008` |

**Важно:**
- Значение `0` означает «не заполнено»/«не применимо» и допустимо для неиспользуемых сторон (как в реальном заказе 000000839).
- Для круглых позиций «без соединения» на активных сторонах используйте `2`.
- Для прямоугольных позиций «без соединения» на активных сторонах используйте `8`.
- Если соединение не применимо для продукта (например, для фланца `12-1-3` или стороны c2/c3/r2/r3), укажите `0`.

### Особенности

- Корневой элемент XML не проверяется модулем — можно использовать любое имя (`<Order>`, `<Root>`, `<Vozd_Obmen>`).
- Для соединений передаётся только **первый символ** значения (`Лев(ХМЛ.Значение, 1)`).
- После загрузки вызывается `ОпределитьТовары()`. Если `ТипПродажи_ДляСайта = "1"`, строка переносится из `Заказы` в `Товары`.
- Для товаров выполняется поиск/создание номенклатуры и характеристик, установка цен.

1. Скопируйте `order_template.xml`.
2. Замените примеры значений на реальные.
3. Сохраните как `Vozd_Obmen.xml` (или другой файл, указанный в поле `ИмяФайлаЗагрузки`).
4. В форме обработки нажмите кнопку загрузки.

## Разбор ошибки из `Ошибка_20260618113438.zip`

### Текст ошибки

```
{ОбщийМодуль.асОбщийМодуль.Модуль(7863)}: Преобразование значения к типу Число не может быть выполнено
```

### Где возникает

Стек вызовов показывает:
1. `Обработка.асВводНовогоЗаказа.Форма.Форма.Форма`, строка 47544 — `ВыполнитьЗагрузкуНаСервере(...)`
2. Строка 47583 — `ОпределениеПараметровИзФайлаЗагрузки()`
3. Строка 32350 — `ТолщинаНовая = асОбщийМодуль.КорректировкаТолщиныМатериалаССайта(Стр.t);`
4. `ОбщийМодуль.асОбщийМодуль.Модуль`, строка 7863 — `Запрос.УстановитьПараметр("ТолщинаССайта", Число(ТолщинаССайта));`

### Причина

В XML-элементе `<thickness>` передано пустое или нечисловое значение. Модуль пытается выполнить `Число(ТолщинаССайта)`, и это падает.

Также в форме есть строка:
```bsl
Стр.t = Число(Стр.t);
```
— она тоже упадёт, если `t` не является числом.

### Решение

Убедитесь, что в XML для каждой строки `<productRow>` элемент `<thickness>` содержит корректное число. Для товаров, где толщина не применима, укажите `0` (не оставляйте пустым):

```xml
<thickness>0</thickness>
```

В шаблоне `order_template.xml` это уже учтено.

## Известные проблемы в коде обработки

### Обозначения всех строк совпадают с последней строкой

**Признак:** после загрузки в колонке «Обозначение» все строки имеют одинаковое значение (например, `Ф160` для всех позиций, даже прямоугольного воздуховода и фланца).

**Причина:** в форме обработки процедура `РедактироватьВсеНаКлиенте_ДляИзмененных()` (строка ~32770) перед вызовом `ОбозначениеПродукцииНаСервере()` не устанавливает заголовочные реквизиты `Объект` из текущей строки. В результате обозначение рассчитывается по параметрам последней обработанной строки.

**Исправление в модуле формы:** в цикле перед вызовом серверной функции нужно скопировать параметры строки в заголовок:

```bsl
Для Каждого Стр Из Объект.Заказы Цикл 
    ЗаполнениеОбъектовБезЗакрытияФормыТуда(Стр);
    Объект.ГруппаПродукцииСтр = Стр.ГруппаПродукцииСтр;
    Объект.СечениеСтр         = Стр.СечениеСтр;
    ОбозначениеПродукцииНаСервере();
    Стр.Обозначение = Объект.Обозначение;
КонецЦикла;
```

Без этого исправления обозначения при загрузке из XML заполняются некорректно и требуют ручного пересчёта (например, пошаговое открытие каждой строки).

## Текущие проблемы и их решение

### 1. «Материала с выбранной толщиной нет»

**Причина:** в `<material>` передан артикул материала или префикс артикула. Функция `НахождениеМатериалаПоТипу` ожидает цифровой код типа материала (`1`, `2`, `3`).

**Решение:** укажите код типа материала: `1` — оцинкованная сталь, `2` — нержавеющая сталь, `3` — чёрная сталь. Толщина подбирается из `<thickness>`.

### 2. Соединения не определяются / ошибки по соединениям

**Причина:** в `<connection_N>` передано некорректное значение, которое не распознаётся функциями `НахождениеСоединениеПродуктаПо_Id_c` / `_r`.

**Решение:**
- Для круглых позиций «без соединения» на активных сторонах используйте `2`.
- Для прямоугольных позиций «без соединения» на активных сторонах используйте `8`.
- Для неиспользуемых/неприменимых сторон укажите `0` (равнозначно пустой ссылке).
- Для активных соединений используйте ID из таблиц выше.
