# Настройка 1C MCP Toolkit для подключения к базе 1С

## Что это даст

После настройки я смогу выполнять запросы к твоей базе 1С напрямую:
- читать справочники (`асПродукция`, `асМатериалы`, `асСоединениеВоздуховодов`);
- получать примеры заказов из `асВводНовогоЗаказа`;
- смотреть реальные значения полей и артикулов.

## Важно: безопасность

Обработка умеет выполнять **произвольные запросы и произвольный код 1С**. Поэтому:

- Лучше настраивать на **копии базы**, а не на рабочей.
- Если настраиваешь на рабочей — используй пользователя только с правами на чтение.
- Не открывай порт `6003` наружу (только localhost или локальная сеть).

---

## Шаг 1. Открыть обработку в 1С

1. Запусти 1С:Предприятие в режиме **толстого клиента**.
2. Войди в базу.
3. Выбери **Файл → Открыть...**
4. Укажи файл: `D:\project\MCP_Toolkit.epf`
5. Откроется форма обработки.

---

## Шаг 2. Выбрать режим «Встроенный сервер»

1. В форме обработки найди переключатель режима.
2. Выбери **«Встроенный сервер»** (этот режим не требует Python/Docker).
3. Укажи порт: `6003` (или другой свободный).
4. (Опционально) укажи ID канала, например `dev`.
5. Нажми **«Запустить сервер»**.

После запуска в форме должна появиться надпись, что сервер запущен, например:

```
Сервер запущен на http://localhost:6003
```

---

## Шаг 3. Проверить, что сервер отвечает

Открой браузер или PowerShell и выполни:

```powershell
Invoke-RestMethod -Uri "http://localhost:6003/api/execute_query" -Method POST -ContentType "application/json" -Body '{"query": "ВЫБРАТЬ 1 КАК Поле"}'
```

Или через curl:

```bash
curl -X POST "http://localhost:6003/api/execute_query" -H "Content-Type: application/json" -d "{\"query\": \"ВЫБРАТЬ 1 КАК Поле\"}"
```

Если всё работает, вернётся JSON с результатом запроса.

---

## Шаг 4. Подключить сервер к Kimi Code CLI

### 4.1. Найти конфигурационный файл MCP

В Kimi Code CLI настройки MCP-серверов обычно находятся в одном из файлов:

- `%APPDATA%\Code\User\settings.json` (если используешь VS Code)
- `%APPDATA%\Code\User\globalStorage\moonshot-ai.kimi-code\settings.json`
- или в файле `.kimi/mcp.json` внутри проекта

### 4.2. Добавить конфигурацию сервера

Добавь в MCP-конфигурацию:

```json
{
  "mcpServers": {
    "onec-mcp-toolkit": {
      "url": "http://localhost:6003/mcp",
      "transport": "http",
      "type": "streamable-http",
      "disabled": false,
      "autoApprove": ["execute_query", "get_metadata"]
    }
  }
}
```

> Если в обработке указал ID канала `dev`, используй URL: `http://localhost:6003/mcp?channel=dev`

### 4.3. Перезапустить Kimi Code CLI

После сохранения конфигурации перезапусти Kimi Code CLI (или VS Code), чтобы сервер появился в списке доступных инструментов.

---

## Шаг 5. Проверить подключение

После перезапуска я должен увидеть новые инструменты, например:

- `execute_query` — выполнение запросов 1С
- `get_metadata` — метаданные конфигурации
- `get_object_by_link` — объект по ссылке

Попробуй попросить меня выполнить тестовый запрос:

```
Выполни запрос к 1С: ВЫБРАТЬ 1 КАК Поле
```

## Альтернатива: REST API без MCP

Если настроить MCP-клиент сложно, можно работать через REST API. Тогда ты сам выполняешь запросы в PowerShell/curl, а присылаешь мне результаты.

Пример запроса к справочнику материалов:

```powershell
$body = @{
    query = "ВЫБРАТЬ асМатериалы.Код, асМатериалы.Наименование, асМатериалы.t ИЗ Справочник.асМатериалы КАК асМатериалы ГДЕ НЕ асМатериалы.ЭтоГруппа"
} | ConvertTo-Json -Depth 10

Invoke-RestMethod -Uri "http://localhost:6003/api/execute_query" -Method POST -ContentType "application/json" -Body $body
```

## Возможные проблемы

| Проблема | Решение |
|----------|---------|
| Порт 6003 занят | Выбери другой порт в обработке и в конфиге |
| Ошибка при открытии обработки | Убедись, что 1С запущена в режиме толстого клиента |
| Сервер не отвечает | Проверь, не блокирует ли брандмауэр порт |
| Нет прав на выполнение запросов | Войди под пользователем с достаточными правами |
