# План поиска правильного формата XML для загрузки заказа в 1С SoftVent

## Проблема
order_example.xml загружается в 1С, но материал не подбирается — толщина 0.7 заменяется на 0.5, а затем пишет «материала с выбранной толщиной: 0.50 нет». Цена = 0. Только строка 3 (прямоугольный) получает цену 222,00, но и у него t=0.50 вместо 0.7.

## Подход 1: Анализ кода модуля загрузки
**Агент:** Анализ_Код_Загрузки (coder)
**Задача:** Глубоко разобрать файл `модуль загрузки с сайта.txt` (49 207 строк), найти:
1. Процедуру `ЗагрузитьДанныеНаСервере` (строка ~32851)
2. Функцию `НахождениеМатериалаПоТипу` — как она ищет материал по артикулу
3. Функцию `ЗаказыtПриИзмененииНаСервере` — как корректирует толщину
4. Почему `material=01-02-01-01-0.70` + `t=0.7` не работает
5. Какие значения ожидаются в `material` — тип, полный артикул или что-то другое

**Выход:** отчёт с конкретными строками кода и выводами.

## Подход 2: Выгрузка реального заказа из 1С в XML
**Агент:** Выгрузка_Заказа_в_XML (coder)
**Задача:**
1. Подключиться к 1С через MCP-сервер (localhost:6003)
2. Выгрузить заказ 000000839 (в котором всё работает) обратно в XML
3. Или найти процедуру выгрузки/экспорта в модуле и посмотреть, какой формат она генерирует
4. Если выгрузка невозможна — сделать запрос к документу и восстановить XML из полей табличной части

**Выход:** XML-файл реального заказа или его структура.

## Интеграция
После получения результатов от обоих агентов — свести воедино и сформировать финальный правильный `order_example.xml`.
