# Руководство по импорту заказов вентиляции в 1С через XML

> Проект: `D:\project`  
> База 1С: `KA_vok-region_test` (1С:Комплексная автоматизация 2.5.22.170)  
> Обработка: `асВводНовогоЗаказа` / `MCP_Toolkit.epf`  
> Детальные заметки исследования см. в `XML_IMPORT_RESEARCH_NOTES.md`.

---

## 1. Общий принцип

Заказ оформляется XML-файлом, который загружается в обработку **«Ввод нового заказа»**. Из обработки формируется документ `асСпецификацияЗаказа`.

**Важно:** перед загрузкой в форме нужно снять галочку **«Авторасчёт»**. Иначе обозначения всех строк считаются по параметрам последней строки (проверено на заказе 000000841 — см. раздел 4).

---

## 2. Структура XML

```xml
<?xml version="1.0" encoding="UTF-8"?>
<Order>
  <numberDate>18.06.2026</numberDate>      <!-- Дата заказа -->
  <numberOrder>0000001</numberOrder>        <!-- Номер заказа -->
  <INN>7721234567</INN>                     <!-- ИНН заказчика -->
  <customer>ООО СтройВент</customer>        <!-- Наименование заказчика -->
  <email>zakaz@stroyvent.example</email>    <!-- Email -->
  <phone>+7 (495) 123-45-67</phone>         <!-- Телефон -->
  <contact>Петров А.В.</contact>            <!-- Контактное лицо -->
  <Tabl>
    <productRow>
      <article>1-1-1</article>
      <name>Воздуховод круглого сечения прямошовный</name>
      <characteristic>D0160D0_L0100L0_</characteristic>
      <quantity>10</quantity>
      <material>1</material>
      <thickness>0.7</thickness>
      <connection_0>2</connection_0>
      <connection_1>2</connection_1>
      <connection_2>0</connection_2>
      <connection_3>0</connection_3>
      <system>Система приточная</system>
      <comment>Цвет RAL 9016</comment>
    </productRow>
  </Tabl>
</Order>
```

## 3. Правила заполнения полей

### 3.1. `<article>` — артикул продукта

Используются **видимые артикулы** из справочника `асПродукция`, например:

- `1-1-1` — Воздуховод круглого сечения прямошовный
- `1-1-2` — Воздуховод круглого сечения спирально-навивной
- `1-2-1` — Воздуховод прямоугольного сечения
- `12-1-3` — Фланец (кр)

Полный список 120 продуктов — в файле `product_designation_patterns.json`.

### 3.2. `<characteristic>` — характеристика

Формат: `ПАРАМЕТР<значение>ПАРАМЕТР_`.

Примеры:

| Артикул | Характеристика | Расшифровка |
|---------|----------------|-------------|
| `1-1-1` | `D0160D0_L0100L0_` | D0=160, L0=100 |
| `1-2-1` | `A0300A0_B0200B0_L0100L0_` | A0=300, B0=200, L0=100 |
| `12-1-3`| `D0160D0_P004P0_` | D0=160, P0=4 |

Значение числа должно быть дополнено нулями (обычно 4 цифры). Парсер ищет подстроку между `ПАРАМЕТР` и `ПАРАМЕТР_`.

Какие параметры нужны для каждого артикула — см. `product_designation_patterns.json`.

### 3.3. `<material>` — тип материала

Передаётся **цифровой код типа материала**, не код справочника:

| Код | Материал |
|-----|----------|
| `1` | Оцинкованная сталь |
| `2` | Нержавеющая сталь |
| `3` | Чёрная сталь |

Точный материал в базе подбирается процедурой `НахождениеМатериалаПоТипу` с учётом толщины.

### 3.4. `<thickness>` — толщина материала

Примеры: `0.5`, `0.7`, `1.0`.

### 3.5. `<connection_N>` — соединения сторон

Передаётся только **первый символ** значения.

**Круглые (`connection_0..connection_3`):**

| Код | Соединение |
|-----|------------|
| `1` | Бандаж |
| `2` | Без соединения |
| `3` | Ниппель |
| `4` | Фланец (кр) |

**Прямоугольные (`connection_0..connection_3`):**

| Код | Соединение |
|-----|------------|
| `5` | Фланец (пр) |
| `6` | Шина / уголок |
| `7` | Рейка |
| `8` | Без соединения |

`0` — сторона не используется.

### 3.6. `<system>` — система вентиляции

Произвольная строка, попадает в `асСистема` строки заказа.

### 3.7. `<comment>` — комментарий

Попадает в `КомментарийКПродукцииИзOnLine`.

## 4. Проблема «Авторасчёт»

### 4.1. Симптомы

При включённой галочке в сохранённом заказе 000000841 обозначения всех строк оказались равны `Ф160` (параметры последней строки):

| Строка | Артикул | Должно быть | В базе |
|--------|---------|-------------|--------|
| 1 | `1-1-1` | `Ф160-100` | `Ф160` |
| 2 | `1-1-2` | `Ф200-100` | `Ф160` |
| 3 | `1-2-1` | `300x200-100` | `Ф160` |
| 4 | `12-1-3`| `Ф160` | `Ф160` |

### 4.2. Решение

Снять галочку **«Авторасчёт»** в форме перед загрузкой.

### 4.3. Проблема автоматизации

Имя реквизита формы, отвечающего за галочку, не найдено в доступных модулях. Без него нельзя передать флаг через XML. Варианты решения:

1. Распаковать `MCP_Toolkit.epf` (`v8unpack`) и найти имя элемента в `асВводНовогоЗаказа.Форма.Форма`.
2. Посмотреть имя реквизита в конфигураторе 1С.
3. Создать собственную обработку-обёртку, которая открывает форму, снимает галочку программно и вызывает загрузку.

## 7. Сопоставление позиций по текстовому описанию

Для загрузки из PDF/Excel/сайта создан скрипт `match_description_to_product.py`.

Он принимает произвольное описание, например:

```bash
python match_description_to_product.py "Воздуховод круглый прямошовный D160 L1000"
```

и возвращает топ подходящих артикулов 1С с извлечёнными размерами:

```text
1. [1-1-1] Воздуховод круглого сечения прямошовный (сечение: round, параметры: D0, L0)
```

Извлекаются:

- диаметр `D0` из `D160`, `DN160`, `Ф160`;
- прямоугольное сечение `A0`/`B0` из `300x200`;
- длина `L0` из `L1000` или последнего крупного числа;
- количество отверстий `P0` для фланцев.

## 8. Загрузка спецификации из CSV/Excel

Для типовых таблиц вида вашей спецификации используется `process_specification_table.py`.

Ожидаемые столбцы:

| Столбец | Пример | Примечание |
|---------|--------|------------|
| `name` | `Воздуховод из оцинкованной стали...` | Описание |
| `size` | `100` или `300x200` | Размер: одиночное число = диаметр круглого, `AxB` = прямоугольное |
| `unit` | `м` / `шт` | `м` → переводится в штуки по стандартной длине |
| `quantity` | `400` | Количество |
| `material` | `оцинкованная` | Опционально, по умолчанию оцинкованная |
| `thickness` | `0.8` | Опционально, по умолчанию 0.8 |
| `ptype` | `elbow` / `tee` / `transition` / `cap` / `flange` / `silencer` | Опционально — тип фасонного изделия; без него строка с неразобранным размером пропускается |

Настройки в начале файла:

```python
DEFAULT_ROUND_ARTICLE = "1-1-2"       # круглый спирально-навивной
DEFAULT_RECT_ARTICLE = "1-2-1"        # прямоугольный
DEFAULT_ROUND_LENGTH_MM = 3000        # длина круглого воздуховода
DEFAULT_RECT_LENGTH_MM = 1250         # длина прямоугольного воздуховода
```

Запуск:

```bash
python process_specification_table.py sample_specification.csv order_from_spec.xml
```

Скрипт автоматически:

- распознаёт круглые и прямоугольные воздуховоды;
- переводит метры в штуки;
- подставляет материал и толщину;
- распознаёт фасонные изделия (отводы, переходы, тройники, заглушки, фланцы, шумоглушители);
- извлекает КСД-адаптеры из строк диффузоров (`4АПН 600x600 + КСД 200`);
- формирует отчёт `order_from_spec_skipped.json` с объяснением, почему пропущены агрегатные строки фасонных изделий и диффузоры без артикула.
